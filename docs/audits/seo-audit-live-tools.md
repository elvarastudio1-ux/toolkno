- 1 live tool has a shorter intro pattern than the rest: `word-counter`.
- Tool pages have strong section structure: H1 in the hero plus H2 sections for What is, How to use, Example, Benefits, and Common questions.
- Every tool has a clean slug and explicit route under `app/tools/<slug>/page.js`.
- Every tool page uses canonical tags and social metadata through `buildToolMetadata(slug)`.
- Every tool page includes schema blocks via `ToolSchemaScripts`, but all pages currently share the same default social image.

## Technical Notes

- `/tools` search results correctly set `noIndex` when a query parameter is present.
- Tool pages are included in `app/sitemap.js` with priority 0.8 and weekly change frequency.
- `app/robots.js` allows public crawling and points to the XML sitemap.
- Tool breadcrumbs in UI and schema stop at `Home > Tools > Tool Name`; category is not included as a breadcrumb node.
- Category pages are marked `force-dynamic` and include server-side `console.log` statements, which is not an indexing blocker but is not ideal for production hygiene.
